Generate the nsbaci installer
When developing, qt get's linked dynamically, so there is no dependency problem. When shipping the executable, you must ship the all the qt environment too. For this, nsbaci uses an installer created with Inno Setup.

If you know how to work with Inno Setup, the .iss is inside installation/.

If you don't, you first need to install Inno Setup, then go into build/ and run:

windeployqt6 nsbaci.exe --release --dir ../build_inno
cd ..
cd installation
echo 'export PATH="$PATH:/c/Program Files (x86)/Inno Setup 6"' >> ~/.bashrc
source ~/.bashrc
ISCC.exe "nsbaci-installer.iss"
The final installable does not carry all the dependencies (some indirect dependencies do not get copied inside the installer). For this, you might need to run the following BEFORE compiling the .iss file (the command above). From the root of the project:
